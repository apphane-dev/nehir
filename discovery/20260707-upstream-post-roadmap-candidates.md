## Recommendation — what to plan next

1. **🔴 `25f4a459` + upstream BarutSRB/OmniWM#437/BarutSRB/OmniWM#299 — transient AX subrole should not seed floating restore.** Small, source-backed guard in `WMController.seedFloatingGeometryIfNeeded`; likely prevents tiled JetBrains/preview windows from reverting to floating after transient popup churn.
2. **🔴 upstream BarutSRB/OmniWM#446 — do not warp the cursor to the center for mouse-click focus.** Nehir has direct evidence: `moveMouseToWindow` always targets `frame.center` and only suppresses while a button is currently pressed; carry a focus-origin or pointer-inside-target guard.
3. **🔴 `679f0ba3` — skip frame-echo window-server queries during active Niri scroll animation.** Tiny performance/stability guard at the AX frame-change seam; Nehir already tracks scroll animations.
4. **🔴 `9abda3d2` — make AX title cache durable until explicit invalidation.** Low-risk hot-path cleanup for workspace-bar reconciliation.
5. **🔴/🟡 `6520c461` — no-op tracked-window readmission guard.** Worth planning after checking Nehir's current refresh/rekey work so it does not fight managed-replacement repairs.
6. **🟡 `2c99f4d0` + `f26ac469` + BarutSRB/OmniWM#449/BarutSRB/OmniWM#444 — compare against Nehir's current offscreen-park/overscroll discoveries.** Treat as evidence for the active Nehir-native park cluster, not a line-port.
7. **🔴 upstream BarutSRB/OmniWM#440 — fullscreen-sized overlay exclusion.** Open a separate Nehir discovery if no current plan owns overlay tools such as screenshot selection windows; use `WindowCapabilityProfile` rather than hard-coded app names if possible.

Explicitly do **not** port in this loop: Quake terminal/Ghostty commits, FoundationModels launch fix, release metadata, platform-floor changes, deletion of config migration machinery, and broad SwiftUI status/menu refactors. Those are either upstream-product-specific or maintainer decisions rather than backport candidates.
