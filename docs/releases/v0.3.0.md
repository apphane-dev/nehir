## Changes

- Improved Niri offscreen window parking behavior and documented display arrangement limitations.
- Added prerelease publishing support with prerelease tags, GitHub prerelease marking, and Homebrew publishing skipped for prereleases.
- Improved hidden window edge parking to avoid wider Workspace switch bleed for strict-clamped app windows.
- Clarified managed command target, border, AX focus, and Niri gesture semantics so hotkeys, visual focus, and snap/no-snap viewport navigation stay aligned.
- Prereleases now publish to nehir@rc Homebrew cask with release notes showing only changes since the previous RC.
- Overhauled debug tracing tools: moved debug commands into a dedicated Debugging & Tracing category, renamed IPC/CLI paths under the `debug` namespace, made Toggle Trace Capture the primary hotkey action, folded trace start/stop scripting into `debug trace toggle <active|inactive>`, added an optional workspace bar trace button, and documented the `invalid_state` IPC error code.
- Released the backing border window when borders are disabled so Nehir does not keep an unnecessary composited surface alive.
- Avoid wrong-display hidden window parking when moving windows to inactive workspaces.
- Add display diagnostics for Dock and monitor arrangement recommendations.
- Improve runtime resize tracing for Niri width commands
- Handle app resize minimums and impossible stacks without resize placeholders

## Notes

Nehir is currently distributed as an unsigned app. macOS may show Gatekeeper warnings until Developer ID signing and notarization are configured.

Nehir requires Accessibility permission to manage windows.
